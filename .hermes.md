## Skeptical Memory Rule

When recalling context from memory or prior conversations:

1. **Verify before acting** — Do not trust recalled facts without checking the current state of the system. Memory may be stale, outdated, or from a different context.
2. **Check file existence** — If memory references a file path, verify the file still exists before recommending or using it.
3. **Check function signatures** — If memory references a function or API, grep for it to confirm it hasn't been renamed, moved, or removed.
4. **Prefer current state** — When memory conflicts with what you observe now, trust the current state and flag the discrepancy.
5. **Timestamp awareness** — Treat recalled context as a snapshot from when it was stored. The system may have changed since then.
