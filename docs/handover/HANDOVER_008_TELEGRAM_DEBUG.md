## 3. Root Causes Identified

### RC-1: Namespace Collision (Hermes `utils/` shadows Vizier `utils/`)
**Impact:** `run_pipeline` and `query_logs` failed with `ModuleNotFoundError`  
**Mechanism:** When Hermes imports Vizier code in-process, Hermes's own `utils/` package takes precedence over Vizier's `utils/` on `sys.path`  
**Fix:** Both `run_pipeline` and `query_logs` handlers rewritten to execute as **subprocesses** (`subprocess.run([sys.executable, "-c", script])`) with clean Python environments  
**File:** `~/.hermes/plugins/vizier_tools/__init__.py`

### RC-2: Stage Tool Output Clobbering
**Impact:** `poster_copy` generated by `generate_poster` tool was overwritten when `image_generate` ran next in the same stage  
**Mechanism:** `stage_output.update(tool_result)` blindly overwrote all keys. When `image_generate` returned `{"poster_copy": ""}`, it erased the real copy  
**Fix:** Changed to merge-without-clobber: skip overwriting a populated key with an empty value  
**File:** `tools/executor.py` (line 422)

### RC-3: Prompt Not Propagated to Stage Tools
**Impact:** Poster image showed generic "Marketing Strategy" content instead of the operator's actual request  
**Mechanism:** `executor.py` used `stage.action` (static YAML text like "Produce poster from artifact spec") as the prompt. The operator's `raw_input` ("Create a Raya poster for DMB") was in `job_context` but never injected into the prompt  
**Fix:** Added `raw_input` extraction from `job_context` and injection into every stage prompt  
**File:** `tools/executor.py` (line 384-389)

### RC-4: No Retry on Transient HTTP Errors
**Impact:** Single 503 from OpenAI crashed the entire pipeline  
**Mechanism:** `httpx.post()` was called once with no retry logic  
**Fix:** Added `_post_with_retry()` with exponential backoff on 502/503/529 (2s, 5s delays, max 2 retries)  
**File:** `utils/call_llm.py`

### RC-5: Text Baked into AI Images
**Impact:** AI-generated poster images contained garbled, unreadable text  
**Mechanism:** `expand_brief()` produced image prompts that didn't explicitly forbid text rendering. FLUX/Kontext models attempted to render text but produced artifacts  
**Fix:** Appended `_NO_TEXT` directive to all visual prompts (anti-drift #49: text is rendered by Typst, never in images)  
**File:** `tools/registry.py` (line 86-92)

### RC-6: Raw HTML Errors Shown to User
**Impact:** Operator saw nginx 503 HTML page in Telegram  
**Mechanism:** Pipeline error messages were passed through without sanitization  
**Fix:** Added regex-based HTML detection and sanitization â€” extracts HTTP status code from HTML, shows clean error message  
**File:** `~/.hermes/plugins/vizier_tools/__init__.py` (line 207-209)

### RC-7: FAL_KEY Leading Whitespace
**Impact:** Potential fal.ai authentication failures  
**Mechanism:** `FAL_KEY= 6e20ed3b-...` (leading space after `=`) in `.hermes/.env`  
**Fix:** Removed leading whitespace  
**File:** `~/.hermes/.env`
